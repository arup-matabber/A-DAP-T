\# Mock Scan Responses



This folder contains example scan responses for A-DAP-T.



These JSON files define the expected response shape for backend and frontend integration before the live scanner is complete.



\## Files



\- vulnerable\_scan\_response.json

\- secured\_scan\_response.json



\## Purpose



The mock responses are used for:



\- frontend dashboard development

\- backend endpoint reference

\- scanner output validation

\- demo consistency

\- team alignment on response fields



\## Required Top-Level Fields



Every scan response should include:



\- project\_name

\- scan\_type

\- safety\_score

\- status

\- summary

\- category\_scores

\- findings

\- graph

\- attack\_replay

\- remediation\_checklist



\## Important Rule



Backend implementation may change internally, but the response shape should stay stable so the frontend does not break.



\## Current Demo Types



\- demo\_vulnerable

\- demo\_secured



\## Notes



These responses are intentionally designed for demonstration and integration. They are not generated by the live scanner yet.
